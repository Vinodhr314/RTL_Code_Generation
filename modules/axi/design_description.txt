Integration context:

- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement AXI4 slave stub with full AW/W/B/AR/R channel signals, 32-bit data, 32-bit address.
FR-002: Slave shall accept AXI4 write/read bursts per channel handshake rules.
FR-003: awready/wready/arready shall follow RTL timing (typically immediate or registered).
FR-004: bresp/rresp shall return OKAY (2'b00) on successful transfers.
FR-005: AXI4-Lite slave stub with 4 32-bit registers (offset 0,4,8,C). Write: accept AW+W, respond with OKAY on B. Read: accept AR, return register

4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single aclk (AXI).
Reset: aresetn active-low.
AXI handshakes: valid/ready per channel; respect TB task sequencing.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
AXI4-Lite slave stub with 4 32-bit registers (offset 0,4,8,C). Write: accept AW+W, respond with OKAY on B. Read: accept AR, return register data on R channel with OKAY.

RTL header notes:
- AXI4-Lite slave stub — 4 x 32-bit registers
- Verilog-2005 synthesizable
- Write address channel
- Write data channel
- Write response channel
- Read address channel
- Read data channel
- 4 x 32-bit register bank

FSM states (from RTL localparams):
- WR_IDLE → WR_DATA → WR_RESP → RD_IDLE → RD_DATA

6. VERIFICATION GUIDANCE
================================================================================
6.1 Reference Testbench: tb/axi_tb.v

6.2 Test Scenarios
  TS-01: AXI write:
  TS-02: 1. Wait for DUT ready BEFORE driving valid (avoids stale-ready issue)
  TS-03: 2. Wait for bvalid BEFORE setting bready (avoids NBA suppression)
  TS-04: Wait for bvalid BEFORE asserting bready
  TS-05: AXI read:
  TS-06: Same approach: wait for DUT arready, then wait for rvalid before rready
  TS-07: Test 1: Write reg 0
  TS-08: Test 2: Read back reg 0
  TS-09: Test 3: Write and read reg 1
  TS-10: Test 4: Write and read reg 2
  TS-11: Test 5: Write and read reg 3
  TS-12: Test 6: reg 0 still holds value
