1. OVERVIEW
================================================================================
General-purpose DMA engine for memory-to-memory transfers..

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS; Split read/write pending flags; avoid mem_wen<=0 same cycle as <=1

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement General-purpose DMA engine for memory-to-memory transfers..
FR-002: Slave shall accept AXI4 write/read bursts per channel handshake rules.
FR-003: awready/wready/arready shall follow RTL timing (typically immediate or registered).
FR-004: bresp/rresp shall return OKAY (2'b00) on successful transfers.
FR-005: On start: copy len bytes from src to dst via mem interface (4 bytes per beat). done+irq pulse when complete.

4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
On start: copy len bytes from src to dst via mem interface (4 bytes per beat). done+irq pulse when complete.

RTL header notes:
- gp_dma — General-purpose DMA stub (Verilog-2005)

FSM states (from RTL localparams):
- IDLE → READ_ST → WRITE_ST → DONE_ST
