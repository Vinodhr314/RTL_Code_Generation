Module: apb_arbiter
Document: Design Specification for Verification (DSV)
Revision: 1.0
Target: 32-bit RISC-V SoC
Protocol/Standard: ARM AMBA APB3

================================================================================
1. OVERVIEW
================================================================================
APB bus arbiter multiplexing multiple masters onto one APB slave bus, parameterized M masters.

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement APB bus arbiter multiplexing multiple masters onto one APB slave bus, parameterized M masters.
FR-002: Slave shall comply with APB3 SETUP/ACCESS two-phase protocol.
FR-003: pready shall assert on ACCESS phase (psel && penable).
FR-004: pslverr shall remain deasserted for valid register accesses.

4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================

RTL header notes:
- APB bus arbiter: M masters to 1 slave, fixed-priority (M0 highest)
- Parameterized M masters; uses flattened port vectors
- Verilog-2005 synthesizable
- Master data signals (flattened pwdata bus: 32b per master)
- Since design_description only lists m_paddr as a multi-master input,
- we simplify: data lines are not listed in spec so we treat pwdata as 0
- and pwrite as a write flag for simplicity in this skeleton.
- APB state

FSM states (from RTL localparams):
- APB_IDLE

APB protocol: Address latched when psel=1 and penable=0. Read/write completes when psel=1, penable=1, pready=1.
