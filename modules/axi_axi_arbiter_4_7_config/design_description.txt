Document: Design Specification for Verification (DSV)

Target: 32-bit RISC-V SoC
Protocol/Standard: ARM AMBA AXI4 / AXI4-Lite

================================================================================
1. OVERVIEW
================================================================================
Simplified 4-master 7-slave AXI4-Lite crossbar arbiter with address decode and round-robin arbitration.

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement Simplified 4-master 7-slave AXI4-Lite crossbar arbiter with address decode and round-robin arbitration.
FR-002: Slave shall accept AXI4 write/read bursts per channel handshake rules.
FR-003: awready/wready/arready shall follow RTL timing (typically immediate or registered).
FR-004: bresp/rresp shall return OKAY (2'b00) on successful transfers.
FR-005: Round-robin arbitration selects one master per cycle. Address decode maps araddr/awaddr upper bits to slave 0-6. s_grant indicates winning m

================================================================================
4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single aclk (AXI).
Reset: aresetn active-low.
AXI handshakes: valid/ready per channel; respect TB task sequencing.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
Round-robin arbitration selects one master per cycle. Address decode maps araddr/awaddr upper bits to slave 0-6. s_grant indicates winning master index. cfg[0] selects round-robin(0) vs priority(1).

RTL header notes:
- AXI crossbar arbiter skeleton: 4 masters, 7 slaves
- Round-robin or priority arbitration; address-decode to slave regions
- Verilog-2005 synthesizable
- Master 0
- Master 1
- Master 2
- Master 3
- Arbitration output

FSM states (from RTL localparams):
- FSM_IDLE

6. VERIFICATION GUIDANCE
================================================================================
6.1 Reference Testbench: tb/axi_axi_arbiter_4_7_config_tb.v

6.2 Test Scenarios
  TS-01: Arbiter AXI write: assert valid, wait for awready (DUT asserts after grant),
  TS-02: then wait for wready (DUT asserts in FSM_WDATA), then wait for bvalid.
  TS-03: Keep valid asserted until the corresponding ready is seen (AXI spec).
  TS-04: Wait for bvalid BEFORE asserting bready (avoids NBA suppression).
  TS-05: Test 1: M0 write to slave 0 region (addr[31:29]=000)
  TS-06: Test 2: M0 read from slave 2 region (addr[31:29]=010 = 0x4000_0000)
  TS-07: Test 3: M1 write to slave 3 region (addr[31:29]=011 = 0x6000_0000)
  TS-08: Test 4: M2 read from slave 5 region (addr[31:29]=101 = 0xA000_0000)
  TS-09: Test 5: Priority mode (cfg[0]=1) M0 write

6.3 Expected Results / Scoreboard

  - rd[2:0] must equal 3'd5

6.4 Pass Criteria
