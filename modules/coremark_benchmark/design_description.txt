2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement Hardware performance counter snapshot for CoreMark runs..
FR-002: Assert busy after start; deassert when done.
FR-003: Present final result on result port when done=1.
FR-004: Pulse irq for one cycle coincident with completion (unless RTL notes otherwise).
FR-005: base_addr and length may be ignored in stub model — document actual RTL behavior.
FR-006: On start: reset cycle_cnt and instret_cnt. Each clock while running increments both. On stop: latch counts (hold until next start).

3. INTERFACE SPECIFICATION
================================================================================
3.1 Port Summary Table
| Port | Direction | Width | Active | Description |
|------|-----------|-------|--------|-------------|
| clk | input | 1 | — | System clock; all sequential logic sampled on rising edge unless noted. |
| rst_n | input | 1 | active-low | Asynchronous active-low reset; deassert synchronously to clk. |
| start | input | 1 | active-high | Level or pulse command to begin accelerator operation. |
| stop | input | 1 | — | Command to halt an in-progress operation. |
| cycle_cnt | output | [31:0] | — | Output port ([31:0]); see functional behavior. |
| instret_cnt | output | [31:0] | — | Output port ([31:0]); see functional behavior. |

3.2 Signal Descriptions
- clk (input, 1, —): System clock; all sequential logic sampled on rising edge unless noted.
- rst_n (input, 1, active-low): Asynchronous active-low reset; deassert synchronously to clk.
- start (input, 1, active-high): Level or pulse command to begin accelerator operation.
- stop (input, 1, —): Command to halt an in-progress operation.
- cycle_cnt (output, [31:0], —): Output port ([31:0]); see functional behavior.
- instret_cnt (output, [31:0], —): Output port ([31:0]); see functional behavior.

================================================================================
4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
On start: reset cycle_cnt and instret_cnt. Each clock while running increments both. On stop: latch counts (hold until next start).
