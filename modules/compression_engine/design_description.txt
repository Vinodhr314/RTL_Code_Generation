================================================================================
1. OVERVIEW
================================================================================
Hardware compressor/decompressor streaming stub (passthrough with 1-cycle latency)..

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement Hardware compressor/decompressor streaming stub (passthrough with 1-cycle latency)..
FR-002: Outputs shall be purely combinational with no clock or reset dependency.
FR-003: All port widths shall match the locked specification exactly.
FR-004: Functional relation: When in_valid: busy=1, next cycle out_valid=1 with in_data/out_last passthrough. Clears out_valid when in_valid deassert.
FR-005: When in_valid: busy=1, next cycle out_valid=1 with in_data/out_last passthrough. Clears out_valid when in_valid deasserts.
