Module: aes
Document: Design Specification for Verification (DSV)
Revision: 1.0
Target: 32-bit RISC-V SoC
Protocol/Standard: FIPS-197 AES (128/256-bit key, ECB stub)

================================================================================
1. OVERVIEW
================================================================================
AES-128/256 block encrypt/decrypt engine (ECB block mode).

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement AES-128/256 block encrypt/decrypt engine (ECB block mode).
FR-002: Accept block/key on valid/start; assert done when digest/ciphertext ready.
FR-003: Output width and algorithm per locked spec and RTL header comments.
FR-004: mode=0 encrypt AES-128 using key[127:0]; mode=1 decrypt. Pulses valid to start; asserts done when block_out ready.

3.2 Signal Descriptions
- clk (input, 1, —): System clock; all sequential logic sampled on rising edge unless noted.
- rst_n (input, 1, active-low): Asynchronous active-low reset; deassert synchronously to clk.
- mode (input, 1, —): Operating mode select.
- key (input, [255:0], —): Cryptographic key material.
- block_in (input, [127:0], —): Crypto/hash input block.
- block_out (output, [127:0], —): Crypto/hash output block.
- valid (input, 1, active-high): Source-valid handshake; data/control is qualified when high.
- done (output, 1, —): Status flag: operation completed successfully.

================================================================================
4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
mode=0 encrypt AES-128 using key[127:0]; mode=1 decrypt. Pulses valid to start; asserts done when block_out ready.

RTL header notes:
- AES-128 ECB encrypt/decrypt (key[127:0], mode: 0=enc 1=dec)

Crypto stub: Algorithm reference values from TB; verify done timing and block_out/digest width.
