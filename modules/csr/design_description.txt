Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement RV32 privileged CSR file: mstatus, mie, mtvec..
FR-002: Reset shall initialize all registered outputs to documented defaults.
FR-003: Interface signaling shall match tb/<name>_tb.v drive/monitor assumptions.
FR-004: csr_op: 00=read, 01=write, 10=set, 11=clear. Addresses: 0x300=mstatus, 0x304=mie, 0x305=mtvec. illegal=1 for unknown addr.

================================================================================
4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
csr_op: 00=read, 01=write, 10=set, 11=clear. Addresses: 0x300=mstatus, 0x304=mie, 0x305=mtvec. illegal=1 for unknown addr.

RTL header notes:
- csr — RV32 privileged CSR file (Verilog-2005)

FSM states (from RTL localparams):
- CSR_MIE → OP_READ → OP_WRITE → OP_SET → OP_CLEAR
