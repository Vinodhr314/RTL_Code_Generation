================================================================================
1. OVERVIEW
================================================================================
APB3 slave register interface for RV32 peripherals.

Integration context:
- Part of the RV32 platform address map and interrupt fabric
- Follows SoC coding_style.md (Verilog-2005, rst_n active-low where clocked)

Verification status: PASS

================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement APB3 slave register interface for RV32 peripherals.
FR-002: Slave shall comply with APB3 SETUP/ACCESS two-phase protocol.
FR-003: pready shall assert on ACCESS phase (psel && penable).
FR-004: pslverr shall remain deasserted for valid register accesses.
FR-005: Four 32-bit registers at offsets 0x0, 0x4, 0x8, 0x0C.
FR-006: APB3 slave with 4 x 32-bit registers. SETUP then ACCESS phase; pready=1 on access.

3. INTERFACE SPECIFICATION
================================================================================
3.1 Port Summary Table
| Port | Direction | Width | Active | Description |
|------|-----------|-------|--------|-------------|
| pclk | input | 1 | — | APB bus clock; sampled on rising edge. |
| presetn | input | 1 | active-low | APB active-low reset; clears registers and deasserts pready. |
| paddr | input | [11:0] | — | APB byte address of the transfer. |

3.2 Signal Descriptions
- pclk (input, 1, —): APB bus clock; sampled on rising edge.
- presetn (input, 1, active-low): APB active-low reset; clears registers and deasserts pready.
- paddr (input, [11:0], —): APB byte address of the transfer.
- psel (input, 1, active-high): APB peripheral select; high during SETUP and ACCESS phases.
- penable (input, 1, active-high): APB enable; low in SETUP, high in ACCESS phase.
- pwrite (input, 1, active-high): APB transfer direction: 1=write, 0=read.
- pwdata (input, [31:0], —): APB write data bus (32-bit).
- prdata (output, [31:0], —): APB read data bus (32-bit).
- pready (output, 1, —): APB slave ready; 1 completes data phase.
- pslverr (output, 1, —): APB slave error; 1 indicates failed transfer.

================================================================================
4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single pclk (APB).
Reset: presetn active-low asynchronous assert, synchronous deassert.
APB timing: SETUP cycle (psel=1, penable=0) then ACCESS cycle (psel=1, penable=1).
DV note: Sample prdata/pready on posedge pclk during ACCESS phase.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
APB3 slave with 4 x 32-bit registers. SETUP then ACCESS phase; pready=1 on access.

RTL header notes:
- APB3 slave — four 32-bit memory-mapped registers

Register map (APB byte addresses):
- 0x000 — REG0 (regs[0])
- 0x004 — REG1 (regs[1])
- 0x008 — REG2 (regs[2])
- 0x00C — REG3 (regs[3])

APB protocol: Address latched when psel=1 and penable=0. Read/write completes when psel=1, penable=1, pready=1.
