================================================================================
2. FUNCTIONAL REQUIREMENTS
================================================================================
FR-001: Module shall implement Synchronous FIFO depth 16, 32-bit data, occupancy count..
FR-002: Writes accepted when wr_en && !full; reads when rd_en && !empty.
FR-003: count shall reflect current occupancy.
FR-004: empty asserted when FIFO contains zero entries; full at depth limit.
FR-005: Standard sync FIFO: write when wr_en&&!full, read when rd_en&&!empty. count=occupancy.

3.2 Signal Descriptions
- clk (input, 1, —): System clock; all sequential logic sampled on rising edge unless noted.
- rst_n (input, 1, active-low): Asynchronous active-low reset; deassert synchronously to clk.
- wr_en (input, 1, active-high): FIFO write enable; qualified when !full.
- rd_en (input, 1, active-high): FIFO read enable; qualified when !empty.
- din (input, [31:0], —): FIFO write data input.
- dout (output, [31:0], —): FIFO read data output.
- full (output, 1, —): FIFO full flag; blocks writes when asserted.
- empty (output, 1, —): FIFO empty flag; blocks reads when asserted.
- count (output, [3:0], —): FIFO occupancy count.

4. TIMING, CLOCKING & RESET
================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
Standard sync FIFO: write when wr_en&&!full, read when rd_en&&!empty. count=occupancy.

RTL header notes:
- fifo — Synchronous FIFO depth 16 (Verilog-2005)

FIFO operation: Push on wr_en when not full; pop on rd_en when not empty. Monitor count/empty/full in scoreboard.
