================================================================================
Clock domain: single clk.
Reset: rst_n active-low (when present).
All sequential outputs update on posedge clk unless RTL specifies otherwise.
Documented latency: 8'd20 clock cycles (localparam CYCLES).

================================================================================
5. FUNCTIONAL BEHAVIOR
================================================================================
On start: FSM performs BFS on hardcoded 4-node adjacency list (0->[1,2], 1->[3], 2->[3], 3->[]). After traversal, asserts done=1, irq=1, result=count of reachable nodes from node 0 (= 4).

RTL header notes:
- bfs — Breadth-First Search accelerator stub (Verilog-2005)
- Hardcoded 4-node adjacency list: 0->[1,2], 1->[3], 2->[3], 3->[]
- BFS from node 0 visits all 4 nodes: 0, 1, 2, 3.
- result = 32'd4 (count of nodes reachable from source node 0)
- FSM: IDLE → BUSY (20 cycles) → DONE; irq pulses one cycle.

FSM states (from RTL localparams):
- IDLE → BUSY_ST → DONE_ST

Hardcoded/stub result constant: RESULT = 32'd4
Busy duration: 8'd20 cycles from start to done.
